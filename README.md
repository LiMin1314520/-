# 文件网盘系统

一个现代化的纯前端文件网盘系统，支持文件上传、下载、列表展示和删除功能。所有文件存储在浏览器本地，无需服务器。

## 功能特性

- 📤 **文件上传** - 支持拖拽上传和点击选择，支持批量上传
- 📥 **文件下载** - 一键下载存储的文件
- 📋 **文件列表** - 清晰展示所有文件，显示文件大小和修改时间
- 🗑️ **文件删除** - 安全删除不需要的文件
- 💾 **本地存储** - 使用 IndexedDB 存储文件，数据持久化
- 🎨 **现代界面** - 采用玻璃态设计，视觉效果出色
- 📱 **响应式设计** - 支持各种屏幕尺寸
- 🔒 **隐私保护** - 文件完全存储在本地浏览器中，不会上传到任何服务器

## 技术栈

- **前端**: HTML5, CSS3, JavaScript (ES6+)
- **存储**: IndexedDB
- **设计**: 玻璃态设计风格

## 使用方法

### 快速开始

1. 直接在浏览器中打开 `templates/index.html` 文件

或者：

2. 使用本地服务器（可选）：
```bash
# 使用 Python
cd file-server
python -m http.server 8000

# 使用 Node.js
npx serve

# 然后访问 http://localhost:8000
```

## 技术细节

### IndexedDB 存储

本应用使用浏览器内置的 IndexedDB 数据库来存储文件：

- **数据库名称**: `FileCloudDB`
- **存储对象**: `files`
- **数据结构**: 包含文件名、大小、类型、修改时间和文件数据

### 优势

- 无需服务器，完全在浏览器中运行
- 数据持久化，关闭浏览器后数据仍然保留
- 支持大文件存储（受浏览器存储限制）
- 隐私保护，文件不会上传到任何服务器

## 注意事项

### 存储限制

IndexedDB 的存储限制取决于浏览器和设备的磁盘空间：

- 通常为浏览器可用磁盘空间的 50%
- 不同浏览器的限制可能不同
- 如果存储空间不足，文件上传将失败

### 浏览器兼容性

支持所有现代浏览器：
- Chrome 25+
- Firefox 10+
- Safari 7+
- Edge 12+

### 数据安全

- 删除文件操作不可恢复，请谨慎操作
- 清除浏览器数据会删除所有存储的文件
- 建议定期备份重要文件

### 性能提示

- 建议上传的文件大小不超过 100MB
- 批量上传大量文件时可能会有性能影响
- 定期清理不需要的文件以释放存储空间

## 配置说明

在 `static/js/app.js` 中可以修改以下配置：

```javascript
const DB_NAME = 'FileCloudDB';      // 数据库名称
const DB_VERSION = 1;               // 数据库版本
const STORE_NAME = 'files';         // 存储对象名称
```

## 常见问题

### Q: 文件存储在哪里？
A: 文件存储在浏览器的 IndexedDB 中，不会上传到任何服务器。

### Q: 能在不同设备间同步文件吗？
A: 不能，本应用是完全本地化的，文件仅存储在当前浏览器的 IndexedDB 中。

### Q: 如何备份文件？
A: 使用下载功能将文件下载到本地即可。

### Q: 清除浏览器数据会怎样？
A: 清除浏览器数据会删除所有存储的文件，请提前备份。
